# Backend Systems Brief Checklist

## High-Risk Mechanisms

- Long-term memory
- Mission judging
- Hidden objectives
- Scoring and ranking
- Paid actions and refunds
- Creator publishing and drafts
- Moderation and age gates
- Deletion, blocking, privacy
- Notifications, events, rewards
- Analytics and audit trails

## Questions To Capture

- What product promise depends on backend behavior?
- What evidence must exist if a user disputes the result?
- What can the user inspect, edit, delete, or appeal?
- What data must be isolated by route, character, mission, user, or creator?
- What state transitions are irreversible?
- What is paid, rate-limited, ranked, public, or safety-sensitive?
- What can fail silently, and what must never fail silently?
- What should be MVP, approximated, experimented with, or scoped out?

## Do Not Decide Yet

- Database vendor
- Schema names
- Exact API routes
- LLM provider
- Vector store
- Queue/event architecture
- Cache strategy
- Deployment topology
